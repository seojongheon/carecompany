# 관리자 홈페이지 콘텐츠 관리 확장 설계

**작성일**: 2026-07-23  
**상태**: 범위 승인됨  
**기준**: 관리자는 콘텐츠와 노출 설정을 관리하며 페이지 레이아웃과 디자인 시스템은 변경하지 않는다.

## 1. 현재 상태 조사

현재 관리자 화면은 사례 생성·수정·삭제, 사진 관리, YouTube 링크, 공개 검토와 공개·비공개 전환까지 구현되어 있다. 반면 아래 고객용 콘텐츠는 컴포넌트 또는 seed 데이터에 고정되어 관리자 화면에서 변경할 수 없다.

| 고객 영역 | 현재 소스 | 필요한 관리자 기능 |
|---|---|---|
| 홈 히어로 | `components/site/home-sections.tsx` | 문구, CTA, 대표 이미지 교체, 이미지 대체 텍스트, 노출 설정 |
| 홈 서비스·사례·과정 섹션 | 컴포넌트와 seed 데이터 | 제목·설명, 노출 여부, 허용 범위 내 순서, 대표 사례 선택 |
| 서비스 4종 | `features/portfolio/data/seed.ts` | 서비스명, 요약, 본문, 대표 이미지, 가격 요약, FAQ, 활성 상태 |
| 가격 안내 | `components/site/pricing-sections.tsx` | 가격 항목 CRUD, 조건, 주의 문구, FAQ, 노출 순서 |

| 사례 | 기존 관리자 기능 | 기존 CRUD·미디어·공개 기능 유지 |

## 2. 선택한 접근법

### 채택: 구조 고정형 콘텐츠 관리

페이지별 고정 슬롯과 반복 항목을 관리자 폼으로 제공한다. 관리자는 텍스트, 이미지, 링크, 가격, 표시 여부와 허용된 목록 순서를 바꿀 수 있다. 섹션 종류 추가, 자유 배치, 컬럼 수 변경, 색상·글꼴 변경은 제공하지 않는다.

이 방식은 현재 화면 구조와 접근성·반응형 품질을 보존하면서 운영자가 자주 바꾸는 내용을 코드 수정 없이 관리할 수 있다.

### 검토했으나 채택하지 않은 방식

- 자유형 페이지 빌더: 유연하지만 반응형·접근성 회귀와 운영 실수 가능성이 커 현재 범위에 맞지 않는다.
- 외부 범용 CMS 우선 연동: 장기적으로 유효하지만 현재 프런트엔드 목업과 저장소 경계 검증보다 연동 비용이 앞선다.

## 3. 관리자 정보 구조

- 대시보드
- 홈페이지 관리
  - 홈
  - 서비스
  - 가격 안내
  - 작업 과정
  - 업체 소개
  - 개인정보처리방침
- 사례 관리
- 미디어 보관함
- 사이트 설정

편집 화면은 `초안 저장 → 미리보기 → 게시` 흐름을 공통으로 사용한다. 저장되지 않은 변경, 저장 실패, 이미지 업로드 실패, 다른 관리자에 의한 변경 충돌을 명확히 표시한다.

## 4. 콘텐츠 경계

### 편집 가능

- 제목, 본문, 보조 문구, 버튼 라벨과 내부·허용된 외부 링크
- 이미지 업로드·교체·삭제, 대체 텍스트, 캡션, 초점 위치
- 가격 항목과 가격 표시 문자열, 포함·제외 범위, 변동 조건, 주의 문구
- 서비스·FAQ·과정 단계·가치 항목 같은 반복 콘텐츠의 추가·수정·삭제·정렬
- 섹션 및 항목 노출 여부, 홈 대표 사례 선택
- 상호, 로고, 연락처, 운영 지역, 사업자 정보, SNS 링크, 기본 SEO

### 편집 불가

- 페이지 템플릿, 섹션 종류, 그리드 구조와 반응형 배치
- 색상, 글꼴, 간격, 컴포넌트 스타일과 애니메이션
- 시스템 필수 내비게이션 경로와 사례의 개인정보 공개 규칙
- 서비스 핵심 분류 키와 사례 데이터 관계를 깨뜨리는 구조 변경

## 5. 안전장치

- 이미지에는 파일 형식·크기 검증, 업로드 진행·재시도, 사용 위치 표시를 제공한다.
- 사용 중인 이미지는 삭제 전에 영향받는 페이지를 알리고 교체 또는 사용 해제 후 삭제하게 한다.
- 링크는 내부 경로 또는 허용된 보안 URL만 저장한다.
- 필수 공개 콘텐츠가 비어 있으면 게시를 막고 필드별 오류를 표시한다.
- 게시 전 데스크톱·모바일 미리보기를 제공하며 공개 사이트에는 게시 버전만 보인다.
- 게시 이력과 최근 게시 버전 복원을 제공한다. 영구 삭제는 확인 절차를 거친다.

## 6. 데이터 흐름과 향후 연동

현재 목업 단계에서는 기존 저장소 경계를 확장해 콘텐츠 초안과 게시 스냅샷을 브라우저에 보존한다. 이미지 파일은 기존 원칙과 같이 세션 미리보기만 제공한다. 실제 운영 단계에서는 인증된 관리자만 원격 콘텐츠와 미디어를 변경하고, 공개 화면은 게시 스냅샷만 읽는다.

## 7. 검증 범위

- 각 콘텐츠 종류의 생성·수정·삭제·정렬·노출 전환
- 초안 저장, 새로고침 복원, 미리보기, 게시, 이전 버전 복원
- 필수값·링크·이미지 검증과 저장/업로드 실패 복구
- 사용 중인 미디어 삭제 방지와 영향 범위 안내
- 게시 전후 고객 화면 반영 및 비공개 초안 비노출
- 390px와 1440px 관리자 편집 흐름, 키보드 조작과 상태 알림

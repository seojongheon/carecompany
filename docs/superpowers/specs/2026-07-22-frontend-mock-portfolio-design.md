# 위생의 기술 프런트엔드 목업 설계

**작성일:** 2026-07-22  
**상태:** 사용자 승인  
**기준 문서:** `hygiene-technology-portfolio-prd.md`  
**시각 참고:** `design/` PDF 및 `figma-ui-agent-master-prompt-ko.md`

## 1. 목표

위생의 기술 포트폴리오의 고객용 사이트와 관리자용 콘텐츠 관리 흐름을 실제로
탐색하고 검증할 수 있는 프런트엔드 목업으로 구현한다. 이번 단계에서는 서버,
Supabase, 인증, 데이터베이스, 원격 스토리지를 연결하지 않는다. 화면과 상호작용은
향후 실제 백엔드 어댑터로 교체할 수 있는 경계를 가진다.

## 2. 권위와 충돌 해결

자료가 충돌하면 PRD를 우선한다. 따라서 서비스 분류는 다음 네 가지로 고정한다.

1. 화장실 청소
2. 에어컨 청소
3. 아파트 유리창 청소
4. 상가 유리창 청소

`design/`의 다른 서비스명, 60장 제한, 로그인 화면처럼 PRD 또는 이번 범위와
충돌하는 내용은 사용하지 않는다. 디자인 자료에서는 색상, 타이포그래피, 간격,
카드, 반응형 구성, 관리자 작업 흐름의 시각 언어만 재사용한다.

## 3. 범위

### 고객용

- 홈, 네 서비스 상세, 전체 사례, 사례 상세, 가격, 작업 과정, 업체 소개,
  개인정보처리방침, 일반 404
- 사례 필터와 명시적 더 보기
- 목록 문맥을 유지하는 URL 기반 빠른보기 모달
- 상세 갤러리 라이트박스
- YouTube 썸네일 우선 지연 로딩 목업
- 모바일 메뉴, 스티키 섹션 탭, 견적 플로팅 버튼의 시각·접근성 상태
- 비공개 사례와 사진의 완전한 고객 비노출

### 관리자

- 로그인 없이 `/admin` 직접 진입
- 대시보드, 사례 목록, 새 사례, 편집, 사진 관리, YouTube 관리, 공개 검토
- 사례 등록·수정·공개/비공개 전환과 자동 저장 표시
- 회당 20장, 사례당 69장 제한과 부분 실패·재시도 목업
- 사진 단계, 순서, 대표 여부, 공개 여부 관리
- 개인정보 체크리스트 완료 전 공개 차단

### 제외

- 인증과 권한 검증
- Supabase와 모든 원격 API
- 데이터베이스와 마이그레이션
- 서버 또는 원격 스토리지 업로드
- 실제 이미지 변환, EXIF 제거, 공개본 복사
- 운영 사이트맵 재검증, 실제 분석·오류 추적·배포 설정
- 견적 폼, 결제, 예약, 고객 계정

## 4. 아키텍처

페이지와 컴포넌트는 데이터 저장 방식을 직접 알지 않는다. `PortfolioRepository`
인터페이스를 통해 서비스와 사례를 읽고 변경한다. 현재 구현은
`LocalStoragePortfolioRepository`가 담당하며, 시드 데이터가 최초 상태를 만든다.
향후 `SupabasePortfolioRepository`로 교체할 때 화면 호출 계약은 유지한다.

공개 읽기 모델과 관리자 편집 모델을 분리한다. 공개 선택자는 공개 상태, 공개 사진,
유효한 대표 이미지만 반환한다. 이를 통해 목업 단계에서도 비공개 데이터가 고객
화면에 나타나지 않는 계약을 테스트할 수 있다.

## 5. 상태와 데이터 흐름

1. 브라우저가 시드 데이터와 저장된 스키마 버전을 확인한다.
2. 저장된 버전이 유효하면 `localStorage` 사례 상태를 복원한다.
3. 관리자가 사례를 수정하면 저장소가 정규화된 JSON만 저장한다.
4. 실제 선택 사진은 `URL.createObjectURL`로 현재 세션에서만 미리보기한다.
5. 새로고침하면 파일 객체는 제거되고 해당 항목은 기본 목업 이미지로 복원된다.
6. 고객 화면은 저장소의 공개 선택자만 사용한다.

이미지 데이터 URL이나 Blob은 브라우저 저장 용량과 안정성 문제 때문에
`localStorage`에 넣지 않는다. 객체 URL은 교체·삭제·언마운트 시 해제한다.

## 6. 주요 상호작용

- 필터는 검색 파라미터와 동기화해 새로고침과 뒤로가기를 지원한다.
- 사례 카드의 소프트 이동은 같은 canonical 경로를 빠른보기 모달로 표시한다.
- 직접 상세 URL 또는 새로고침은 독립 상세 화면으로 표시한다.
- 모달을 닫으면 필터, 스크롤, 포커스가 원래 목록 문맥으로 돌아간다.
- 라이트박스는 상세 화면에서만 열고 빠른보기 위에는 중첩하지 않는다.
- 공개 검토는 필수 정보, 대표 이미지, 공개 사진, 개인정보 확인을 검증한다.
- 업로드 실패는 성공 항목을 보존하고 실패 항목에만 재시도를 제공한다.

## 7. 오류와 복구

- 손상되거나 구버전인 로컬 데이터는 별도 키로 백업 후 시드 데이터로 복원한다.
- 저장 실패 시 현재 화면 입력을 유지하고 재시도 안내를 표시한다.
- 파일 제한 또는 형식 오류는 선택 단계에서 차단하고 구체적인 다음 행동을 제공한다.
- 고객용 미디어 오류는 레이아웃을 유지하는 대체 UI로 처리한다.
- 존재하지 않거나 비공개인 사례는 동일한 일반 404로 처리한다.

## 8. 테스트 전략

- 단위 테스트: 저장소 계약, 마이그레이션, 공개 선택자, 필터, 업로드 제한,
  YouTube 파서, 공개 체크리스트
- 컴포넌트 테스트: 카드, 필터, 모달 포커스, 업로드 상태, 폼 오류
- E2E: 홈부터 사례 탐색, 모달 히스토리, 상세 라이트박스, 관리자 등록·분류·공개,
  새로고침 복원, 비공개 비노출, 20/69장 제한
- 시각 검토: 390px와 1440px, 주요 PDF 레이아웃, 텍스트 잘림, 키보드 포커스
- 완료 게이트: lint, typecheck, test, e2e, build

## 9. 후속 연동 문서

`docs/project/deferred-backend-integration.md`를 구현과 함께 유지한다. 각 항목에는
현재 목업 동작, 실제 연동 목표, 교체 파일, 데이터 이전, 보안·개인정보 검증,
테스트, 배포 전 완료 체크를 기록한다. 최소 범위는 다음과 같다.

- Supabase Auth와 관리자 접근 제어
- Postgres 스키마, 마이그레이션, 생성 타입
- RLS와 Storage 정책
- TUS 업로드, 공개본 생성, HEIC 변환, EXIF 제거
- 서버 검증, 캐시 무효화, 사이트맵과 메타데이터
- Sentry, Analytics, Search Console, 네이버 검색 등록
- 실제 로고, 연락처, 사업자 정보, 가격, 사례 콘텐츠
- 목업 데이터 폐기 또는 운영 데이터 마이그레이션

## 10. 승인된 결정

- PRD가 모든 충돌에서 우선한다.
- 목업 데이터는 `localStorage`에 유지한다.
- 선택 사진은 세션 미리보기만 제공하고 새로고침 후 목업 이미지로 복원한다.
- `/admin`은 인증 없이 직접 진입한다.
- 현재 어댑터와 향후 연동 작업을 명시적으로 문서화한다.

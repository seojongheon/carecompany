# Feature Specification: 관리자 홈페이지 콘텐츠 관리

**Feature Branch**: `not-applicable-no-git-repository`

**Created**: 2026-07-23

**Status**: Approved scope

**Input**: 기존 사례 관리자 기능을 확장하여 관리자가 랜딩페이지 이미지, 가격, 서비스 안내, 작업 과정, 업체 소개, 공통 사이트 정보와 사례를 한 곳에서 생성·수정·삭제·게시할 수 있게 한다. 페이지 레이아웃과 디자인 자체는 편집 범위에서 제외한다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 홈과 공통 사이트 정보 관리 (Priority: P1)

관리자는 개발자 도움 없이 홈의 핵심 문구와 이미지, 버튼, 연락처와 푸터 정보를 수정하고 고객에게 공개할 수 있다.

**Why this priority**: 첫 화면과 연락 정보는 영업 메시지와 신뢰에 직접 영향을 주며 변경 빈도가 높다.

**Independent Test**: 홈 히어로 문구와 대표 이미지를 교체하고 모바일·데스크톱 미리보기를 확인한 뒤 게시하여 고객 화면에 반영되는지 검증한다.

**Acceptance Scenarios**:

1. **Given** 관리자가 홈 편집 화면을 열었을 때, **When** 히어로 제목·설명·CTA·대표 이미지를 수정하고 초안을 저장하면, **Then** 공개 사이트는 바뀌지 않고 관리자 미리보기에서만 변경을 확인할 수 있다.
2. **Given** 필수 문구와 이미지 대체 텍스트가 유효할 때, **When** 관리자가 게시하면, **Then** 홈의 게시 버전이 한 번에 갱신된다.
3. **Given** 관리자가 로고·연락처·사업자 정보·외부 링크를 수정했을 때, **When** 사이트 설정을 게시하면, **Then** 해당 정보를 사용하는 헤더·푸터·공통 CTA에 일관되게 반영된다.

---

### User Story 2 - 서비스와 가격 관리 (Priority: P1)

관리자는 네 서비스의 설명·대표 이미지·FAQ와 가격 항목을 관리하고 노출 순서를 조정한다.

**Why this priority**: 서비스 범위와 가격 안내는 고객의 비교와 문의 결정에 직접 영향을 준다.

**Independent Test**: 한 서비스의 설명과 대표 이미지를 수정하고 가격 항목을 추가·수정·삭제·정렬한 뒤 게시하여 서비스와 가격 화면의 동시 반영을 검증한다.

**Acceptance Scenarios**:

1. **Given** 서비스 편집 화면에서, **When** 관리자가 요약·본문·대표 이미지·가격 요약·FAQ를 수정하면, **Then** 미리보기에서 연결된 서비스 화면을 확인할 수 있다.
2. **Given** 가격 항목이 여러 개 있을 때, **When** 관리자가 항목을 추가·수정·삭제·정렬하면, **Then** 저장된 순서대로 가격 안내와 해당 서비스 요약에 표시된다.
3. **Given** 공개 중인 서비스 또는 가격 항목을 숨기거나 삭제하려 할 때, **When** 작업을 확인하면, **Then** 영향받는 고객 화면이 안내되고 확인 전에는 공개 상태가 바뀌지 않는다.

---

### User Story 3 - 안내 페이지 콘텐츠 관리 (Priority: P2)

관리자는 작업 과정, 업체 소개, 개인정보처리방침의 텍스트·이미지·반복 항목과 노출 상태를 관리한다.

**Why this priority**: 신뢰 정보와 운영 정책을 최신 상태로 유지하되 핵심 영업 콘텐츠 이후에 독립적으로 제공할 수 있다.

**Independent Test**: 작업 과정 단계를 재정렬하고 업체 소개 이미지를 교체하며 정책 본문을 수정한 뒤 각각 미리보기·게시한다.

**Acceptance Scenarios**:

1. **Given** 작업 과정에 여러 단계가 있을 때, **When** 관리자가 단계를 추가·수정·삭제·정렬하면, **Then** 고객 화면이 게시 순서와 번호를 연속적으로 표시한다.
2. **Given** 업체 소개 편집 화면에서, **When** 소개 문구·가치 항목·이미지·CTA를 수정하면, **Then** 고정된 페이지 레이아웃 안에서 변경 내용이 표시된다.
3. **Given** 개인정보처리방침 초안이 수정되었을 때, **When** 필수 검토 확인 없이 게시를 시도하면, **Then** 게시가 차단되고 필요한 확인 항목이 안내된다.

---

### User Story 4 - 미디어 자산 관리 (Priority: P2)

관리자는 홈페이지와 안내 페이지에서 사용할 이미지를 업로드하고 재사용하며, 안전하게 교체하거나 삭제한다.

**Why this priority**: 이미지가 여러 페이지에서 재사용될 수 있으므로 중복 업로드와 잘못된 삭제를 줄여야 한다.

**Independent Test**: 이미지 한 장을 업로드해 홈과 업체 소개에서 사용한 뒤 교체하고, 사용 중인 원본의 삭제 차단과 사용 위치 안내를 확인한다.

**Acceptance Scenarios**:

1. **Given** 유효한 이미지 파일을 선택했을 때, **When** 업로드하면, **Then** 진행·성공·실패 상태와 재시도 행동이 표시된다.
2. **Given** 이미지가 하나 이상의 공개 콘텐츠에서 사용 중일 때, **When** 관리자가 삭제를 시도하면, **Then** 사용 위치가 표시되고 교체 또는 사용 해제 전까지 삭제할 수 없다.
3. **Given** 이미지가 어디에서도 사용되지 않을 때, **When** 관리자가 삭제를 확인하면, **Then** 자산 목록과 향후 선택 목록에서 제거된다.

---

### User Story 5 - 안전한 게시와 복원 (Priority: P1)

관리자는 저장 중인 초안과 고객에게 보이는 게시 버전을 구분하고, 게시 전 검증과 이전 버전 복원으로 실수를 줄인다.

**Why this priority**: 전체 홈페이지를 수정할 수 있는 권한은 잘못된 공개의 영향도 크므로 사례 공개와 같은 수준의 안전장치가 필요하다.

**Independent Test**: 유효하지 않은 초안의 게시 차단, 유효한 초안의 게시, 새로고침 후 복원, 최근 게시 버전으로의 복원을 연속해서 검증한다.

**Acceptance Scenarios**:

1. **Given** 필수값·링크·이미지 설명 중 하나가 유효하지 않을 때, **When** 게시를 시도하면, **Then** 게시가 차단되고 각 오류 위치로 이동할 수 있다.
2. **Given** 유효한 초안이 있을 때, **When** 게시하면, **Then** 관련 고객 화면에는 완전한 새 게시 버전만 표시되고 저장 중간 상태는 노출되지 않는다.
3. **Given** 이전 게시 버전이 존재할 때, **When** 관리자가 복원을 선택하고 확인하면, **Then** 선택한 버전이 새 초안으로 복제되어 미리보기 후 다시 게시할 수 있다.

### Edge Cases

- 현재 편집 중인 콘텐츠를 다른 관리자 또는 세션이 먼저 변경하면 충돌을 알리고, 덮어쓰기 전에 비교 또는 새로고침을 요구한다.
- 이미지 업로드 일부만 실패하면 성공한 항목은 유지하고 실패한 항목만 재시도할 수 있다.
- 반복 항목을 모두 숨기거나 삭제해 필수 섹션이 비게 되면 게시를 차단한다.
- 외부 링크가 허용되지 않은 형식이거나 안전하지 않은 주소이면 저장 전에 거부한다.
- 게시 도중 실패하면 기존 게시 버전을 유지하고 초안과 재시도 행동을 보존한다.
- 비활성 서비스에 연결된 공개 사례는 관리자에게 영향을 경고하며 별도 확인 없이 숨기지 않는다.
- 세션 전용 이미지 미리보기가 새로고침으로 사라지면 텍스트 초안은 유지하고 이미지 재선택 필요 상태를 표시한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 관리자는 홈 히어로, 홈 섹션 문구, CTA, 대표 이미지와 대체 텍스트를 수정할 수 있어야 한다.
- **FR-002**: 관리자는 홈의 고정된 섹션을 표시하거나 숨기고, 허용된 섹션과 항목의 노출 순서를 변경할 수 있어야 한다.
- **FR-003**: 관리자는 서비스명, 요약, 본문, 대표 이미지, 가격 요약, FAQ와 활성 상태를 관리할 수 있어야 한다.
- **FR-004**: 네 서비스의 핵심 분류 키와 사례 연결 관계는 일반 콘텐츠 편집으로 변경할 수 없어야 한다.
- **FR-005**: 관리자는 가격 항목, 표시 가격, 단위·범위 문구, 포함·제외 내용, 변동 조건과 주의 문구를 생성·수정·삭제·정렬할 수 있어야 한다.
- **FR-006**: 가격 데이터는 개별 사례의 과거 작업비와 연결되지 않아야 한다.
- **FR-007**: 관리자는 작업 과정 단계, 업체 소개 항목, 개인정보처리방침 본문을 관리할 수 있어야 한다.
- **FR-008**: 관리자는 로고, 상호, 서비스 지역, 연락처, 사업자 정보, SNS·외부 링크와 기본 검색·공유 정보를 관리할 수 있어야 한다.
- **FR-009**: 관리자는 사이트용 이미지를 업로드·선택·교체·삭제하고 대체 텍스트, 캡션과 초점 위치를 관리할 수 있어야 한다.
- **FR-010**: 사용 중인 이미지 삭제 전 시스템은 사용 위치와 영향을 표시하고 참조가 남아 있는 삭제를 차단해야 한다.
- **FR-011**: 시스템은 이미지 형식·용량과 링크 형식을 검증하고 실패 원인과 복구 행동을 제공해야 한다.
- **FR-012**: 모든 콘텐츠 편집은 초안 저장과 게시를 구분해야 하며 고객은 게시 버전만 볼 수 있어야 한다.
- **FR-013**: 관리자는 게시 전에 데스크톱과 모바일 미리보기를 확인할 수 있어야 한다.
- **FR-014**: 시스템은 필수 콘텐츠, 이미지 대체 텍스트, 링크와 노출 규칙이 유효하지 않으면 게시를 차단해야 한다.
- **FR-015**: 게시 작업은 관련된 모든 고객 화면에 완전한 한 버전으로 반영되어야 한다.
- **FR-016**: 시스템은 콘텐츠별 최근 수정 시각, 게시 시각과 수정 주체를 기록해야 한다.
- **FR-017**: 관리자는 최근 게시 버전을 확인하고 선택한 버전을 새 초안으로 복원할 수 있어야 한다.
- **FR-018**: 삭제·숨김·비활성화처럼 고객 노출에 영향을 주는 동작은 영향 범위와 확인 절차를 제공해야 한다.
- **FR-019**: 저장·업로드·게시 실패 시 입력과 성공한 작업을 보존하고 실패한 작업만 재시도할 수 있어야 한다.
- **FR-020**: 기존 사례 CRUD, 사진·YouTube 관리와 공개 검토 기능은 동일하게 유지되어야 한다.
- **FR-021**: 관리자 콘텐츠 기능은 모바일과 데스크톱에서 동일한 핵심 작업을 완료할 수 있어야 한다.
- **FR-022**: 관리자 화면은 페이지 템플릿, 색상, 글꼴, 그리드 또는 임의 컴포넌트 배치를 변경하는 기능을 제공하지 않아야 한다.
- **FR-023**: 비공개 초안, 삭제된 콘텐츠와 세션 전용 미디어는 고객 화면과 검색 노출 대상에서 제외되어야 한다.

### Key Entities

- **사이트 설정**: 상호, 로고, 연락처, 운영 지역, 사업자 정보, 외부 링크와 기본 검색·공유 정보를 나타낸다.
- **페이지 콘텐츠**: 홈, 작업 과정, 업체 소개, 개인정보처리방침 등 페이지별 고정 슬롯과 게시 상태를 나타낸다.
- **콘텐츠 항목**: 가격, FAQ, 과정 단계, 가치 항목처럼 페이지 안에서 추가·수정·삭제·정렬할 수 있는 반복 항목이다.
- **서비스 콘텐츠**: 네 서비스의 고정 식별자와 편집 가능한 이름·설명·대표 이미지·가격 요약·FAQ·활성 상태를 나타낸다.
- **사이트 미디어**: 홈페이지용 이미지와 대체 텍스트, 캡션, 초점 위치, 사용 위치와 처리 상태를 나타낸다.
- **콘텐츠 버전**: 초안 또는 게시 스냅샷, 수정 주체와 시각, 검증 결과와 복원 관계를 나타낸다.
- **포트폴리오 사례**: 기존 사례 관리 대상이며 서비스 콘텐츠와 연결되지만 가격 항목과는 연결되지 않는다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 관리자는 홈 히어로 문구와 대표 이미지 교체부터 미리보기·게시까지 5분 이내에 완료할 수 있다.
- **SC-002**: 관리자는 서비스 가격 항목 하나를 추가·수정·정렬·게시하는 작업을 3분 이내에 완료할 수 있다.
- **SC-003**: 게시 검증에서 필수값 누락, 대체 텍스트 누락, 잘못된 링크와 사용 중 미디어 삭제의 고객 노출 사고는 0건이다.
- **SC-004**: 게시 전후 비교에서 공개 사이트에 비공개 초안 또는 부분 게시 상태가 노출되는 경우는 0건이다.
- **SC-005**: 핵심 관리자 콘텐츠 작업은 390px와 1440px에서 가로 스크롤 없이 완료 가능하고 키보드만으로도 수행할 수 있다.
- **SC-006**: 저장·업로드·게시 실패 시 이미 입력한 유효 콘텐츠와 성공한 항목의 보존율은 100%다.
- **SC-007**: 비기술 운영자 대상 사용성 확인에서 90% 이상이 별도 개발 지원 없이 홈, 가격, 업체 소개 중 하나를 수정하고 게시한다.
- **SC-008**: 이전 게시 버전 복원 작업은 3분 이내에 완료되며 복원 전 현재 게시 버전은 유지된다.

## Assumptions

- 관리자는 콘텐츠와 노출 설정만 변경하며 페이지 구조와 디자인 시스템은 변경하지 않는다.
- 네 서비스 분류는 유지되며 이름과 설명은 편집할 수 있지만 내부 식별자와 사례 연결은 보호된다.
- 현재 프런트엔드 목업에서는 콘텐츠 상태를 브라우저에 보존하고 선택 이미지 파일은 세션 미리보기로만 유지한다.
- 실제 운영 단계에서는 기존 계획의 인증, 원격 저장, 권한과 게시 이력을 사용하며 이 기능명세는 특정 기술을 강제하지 않는다.
- 사이트 이미지와 사례 이미지는 용도와 공개 검토 기준이 달라 논리적으로 구분해 관리한다.
- 개인정보처리방침은 운영자가 법률 검토한 문안을 입력하며 시스템은 법률 적합성을 판단하지 않는다.
- 관리자 편집 기능 추가로 기존 고객 URL, 사례 공개 규칙과 개별 사례 가격 비노출 원칙은 바뀌지 않는다.

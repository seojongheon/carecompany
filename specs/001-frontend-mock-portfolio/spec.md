# Feature Specification: 위생의 기술 프런트엔드 목업 포트폴리오

**Feature Branch**: `not-applicable-no-git-repository`

**Created**: 2026-07-22

**Status**: Approved scope

**Input**: 고객용 청소 포트폴리오와 모바일 우선 관리자 기능을 PRD 및 디자인
자료에 맞춰 구현하되, 서버·데이터베이스·인증·원격 스토리지 연동 없이 동작하는
프런트엔드 목업으로 제공한다. 임시 대체 사항과 향후 실제 연동 작업을 문서화한다.

## Clarifications

### Session 2026-07-22

- Q: PRD와 디자인 자료가 충돌하면 어떤 자료를 우선하는가? → A: PRD를 우선한다.
- Q: 목업 사례와 상태를 새로고침 뒤에도 유지하는가? → A: `localStorage`로 유지한다.
- Q: 실제 선택 사진을 브라우저 영구 저장소에 보관하는가? → A: 세션 미리보기만
  제공하고 새로고침 뒤에는 기본 목업 이미지로 복원한다.
- Q: 관리자 화면에 로그인 단계를 포함하는가? → A: 이번 범위에서는 제외하고
  `/admin`으로 직접 진입한다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 서비스와 공개 사례 탐색 (Priority: P1)

고객은 홈에서 위생의 기술이 제공하는 네 가지 청소 서비스를 즉시 이해하고,
자신에게 맞는 서비스 페이지에서 공개된 현장 사례를 빠르게 탐색한다.

**Why this priority**: 사이트의 핵심 영업 가치이며, 고객이 업체의 작업 범위와 실제
결과를 판단하는 첫 흐름이다.

**Independent Test**: 홈에서 네 서비스 중 하나를 선택하고, 서비스 고유 설명과
필터링된 공개 사례 목록을 확인하며, 더 보기로 다음 사례 묶음을 불러올 수 있다.

**Acceptance Scenarios**:

1. **Given** 고객이 홈에 진입했을 때, **When** 서비스 영역을 확인하면, **Then**
   화장실·에어컨·아파트 유리창·상가 유리창 청소가 각각 독립 카드로 표시된다.
2. **Given** 공개 및 비공개 사례가 함께 존재할 때, **When** 고객이 서비스 페이지를
   열면, **Then** 공개 사례만 목록·개수·필터 결과에 포함된다.
3. **Given** 사례가 첫 표시 수량보다 많을 때, **When** 고객이 `사례 더 보기`를
   선택하면, **Then** 기존 목록을 유지하면서 다음 공개 사례 묶음이 추가된다.
4. **Given** 고객이 필터를 선택했을 때, **When** 새로고침 또는 뒤로가기를 하면,
   **Then** 선택한 조건이 URL과 화면에 유지된다.

### User Story 2 - 사례를 빠르게 보고 깊게 확인 (Priority: P1)

고객은 목록의 현재 위치를 잃지 않고 빠른보기로 사례의 핵심 사진과 정보를 확인한
뒤, 필요하면 독립 상세 화면에서 전체 작업 과정과 미디어를 살펴본다.

**Why this priority**: 많은 사례를 부담 없이 비교하면서도 검색·공유 가능한 상세
콘텐츠를 제공하는 제품의 핵심 차별점이다.

**Independent Test**: 서비스 목록에서 사례 카드를 열어 빠른보기, 이전·다음 이동,
닫기, 독립 상세 이동, 갤러리 확대까지 한 흐름으로 검증할 수 있다.

**Acceptance Scenarios**:

1. **Given** 고객이 사례 목록을 보고 있을 때, **When** 사례 카드를 선택하면,
   **Then** 주소는 사례 경로로 바뀌고 목록 위에 빠른보기가 열린다.
2. **Given** 빠른보기가 열려 있을 때, **When** 브라우저 뒤로가기를 실행하면,
   **Then** 목록을 떠나지 않고 모달이 닫히며 기존 필터·스크롤·포커스가 복원된다.
3. **Given** 사례 경로를 직접 열거나 새로고침했을 때, **When** 화면이 표시되면,
   **Then** 모달이 아닌 독립 상세 화면이 열린다.
4. **Given** 상세 화면에 공개 사진이 있을 때, **When** 사진을 선택하면, **Then**
   번호·캡션·이전·다음·닫기를 제공하는 라이트박스가 열린다.
5. **Given** 비공개 또는 존재하지 않는 사례 경로일 때, **When** 고객이 접근하면,
   **Then** 상태를 노출하지 않는 동일한 일반 404가 표시된다.

### User Story 3 - 관리자 사례 초안 작성과 복원 (Priority: P2)

관리자는 로그인 과정 없이 관리자 화면에 들어가 새 비공개 사례를 만들고 기본
정보를 수정하며, 브라우저를 새로고침한 뒤에도 텍스트와 공개 상태를 이어서 관리한다.

**Why this priority**: 실제 백엔드 연결 전 관리자 작업 흐름과 정보 구조를 검증하는
데 필요한 최소 관리 기능이다.

**Independent Test**: `/admin`에 직접 진입해 새 사례를 만들고 제목·서비스·지역을
수정한 뒤 새로고침해 동일한 초안이 복원되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 사용자가 `/admin`에 접근할 때, **When** 화면이 열리면, **Then** 로그인
   또는 권한 확인 없이 관리자 대시보드가 표시된다.
2. **Given** 관리자가 새 사례의 최소 정보를 입력했을 때, **When** 임시 저장하거나
   자동 저장이 완료되면, **Then** 사례는 기본적으로 비공개 상태로 목록에 추가된다.
3. **Given** 저장된 목업 사례가 있을 때, **When** 동일한 브라우저에서 새로고침하면,
   **Then** 사례 텍스트·분류·정렬·공개 상태가 복원된다.
4. **Given** 저장 데이터가 손상되었을 때, **When** 관리자 화면을 다시 열면, **Then**
   사용 가능한 기본 사례로 복구되고 복구 안내가 표시된다.

---

### User Story 4 - 사진·영상 정리와 공개 검토 (Priority: P2)

관리자는 사진 선택과 업로드 상태를 체험하고, 사진을 작업 단계별로 분류하며,
YouTube 링크와 개인정보 검토 항목을 확인한 후 사례를 공개한다.

**Why this priority**: 모바일 현장 업무의 핵심이며, 공개 자료의 품질과 개인정보
경계를 검증한다.

**Independent Test**: 사진을 선택해 제한·진행·실패 상태를 확인하고, 단계와 대표
사진을 지정한 뒤 공개 체크리스트를 완료해 고객 화면 노출까지 검증할 수 있다.

**Acceptance Scenarios**:

1. **Given** 관리자가 사진을 선택할 때, **When** 한 번에 21장 이상을 선택하면,
   **Then** 업로드 시작 전에 최대 20장 제한과 다시 선택 행동이 안내된다.
2. **Given** 사례에 69장이 등록되어 있을 때, **When** 추가를 시도하면, **Then**
   추가 동작이 차단되고 기존 사진을 삭제한 뒤 추가할 수 있음을 안내한다.
3. **Given** 일부 파일이 실패했을 때, **When** 업로드 상태를 확인하면, **Then**
   성공 항목은 유지되고 실패 항목에만 재시도와 제거 행동이 제공된다.
4. **Given** 사용자가 선택한 실제 사진이 세션에 있을 때, **When** 새로고침하면,
   **Then** 텍스트 메타데이터는 유지되고 사진은 기본 목업 이미지로 안전하게 복원된다.
5. **Given** 공개 필수 정보나 개인정보 확인이 부족할 때, **When** 공개를 시도하면,
   **Then** 공개가 차단되고 부족한 조건이 구체적으로 표시된다.
6. **Given** 모든 공개 조건이 충족되었을 때, **When** 관리자가 공개하면, **Then**
   사례와 공개 사진만 고객 목록·상세·관련 사례에 표시된다.

---

### User Story 5 - 안내 콘텐츠와 반응형·접근성 이용 (Priority: P3)

고객과 관리자는 모바일과 데스크톱에서 가격, 작업 과정, 업체 정보, 오류 상태를
읽고 주요 기능을 키보드와 터치로 이용한다.

**Why this priority**: 핵심 사례 흐름을 보완하고 실제 출시 전에 필요한 콘텐츠 구조와
품질 기준을 검증한다.

**Independent Test**: 390px 및 1440px 화면에서 주요 페이지를 탐색하고, 키보드만으로
메뉴·필터·모달·라이트박스·관리자 폼을 사용할 수 있다.

**Acceptance Scenarios**:

1. **Given** 고객이 가격 안내를 열었을 때, **When** 서비스별 정보를 읽으면, **Then**
   개별 사례의 과거 가격이 아닌 교체 가능한 기준 가격과 변동 조건이 표시된다.
2. **Given** 키보드 사용자가 주요 흐름을 이용할 때, **When** 메뉴·카드·모달을
   이동하면, **Then** 포커스가 보이고 열기·닫기 후 올바른 위치로 이동한다.
3. **Given** 모바일 화면에서 메뉴 또는 모달이 열렸을 때, **When** 플로팅 견적
   버튼을 확인하면, **Then** 중요한 콘텐츠와 겹치지 않으며 오버레이 중에는 숨겨진다.
4. **Given** 사용자가 동작 감소를 선호할 때, **When** 전환이 발생하면, **Then**
   불필요한 이동·확대 애니메이션 없이 상태가 바뀐다.

### Edge Cases

- 필터 결과가 없으면 숨겨진 사례 수를 암시하지 않고 필터 초기화를 제공한다.
- 공개 사례에 공개 사진이 없으면 고객에게 노출하지 않고 관리자에게 공개 조건 오류를
  표시한다.
- 대표 사진이 비공개로 바뀌면 공개 가능한 대체 대표 사진을 요구한다.
- 중복 파일명은 허용하되 화면 항목은 고유하게 구분한다.
- 허용되지 않은 파일 형식 또는 손상된 파일은 다른 성공 파일에 영향을 주지 않는다.
- YouTube 일반·단축·Shorts 이외의 주소는 저장 전에 거부한다.
- 빠른보기에서 필터로 제외된 사례를 이전·다음 대상으로 포함하지 않는다.
- 미디어가 하나뿐인 그룹은 불필요한 이전·다음 컨트롤을 표시하지 않는다.
- 브라우저 저장 공간 사용이 실패하면 현재 입력을 유지하고 저장 실패를 안내한다.
- 목업 데이터 스키마 버전이 바뀌면 호환 가능한 값만 이전하고 나머지는 기본값으로
  복구한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 홈은 PRD의 네 서비스를 각각 한 번의 선택으로 열 수 있어야 한다.
- **FR-002**: 각 서비스는 고유 경로·제목·설명·사례 필터를 가져야 한다.
- **FR-003**: 고객 화면은 공개 사례와 공개 사진만 표시해야 한다.
- **FR-004**: 서비스 및 전체 사례 목록은 필터와 명시적 더 보기를 제공해야 한다.
- **FR-005**: 필터 상태는 공유·새로고침·뒤로가기가 가능한 주소 상태를 가져야 한다.
- **FR-006**: 사례 선택은 목록 문맥을 유지하는 빠른보기를 열어야 한다.
- **FR-007**: 빠른보기는 사례 경로, 닫기, 이전·다음, 포커스 복귀를 지원해야 한다.
- **FR-008**: 사례 경로의 직접 접근은 독립 상세 화면을 표시해야 한다.
- **FR-009**: 상세 화면은 공개 미디어를 작업 전·과정·후·세부로 구분해야 한다.
- **FR-010**: 상세 사진은 키보드와 터치를 지원하는 라이트박스로 확대할 수 있어야 한다.
- **FR-011**: 공개 YouTube 항목은 재생 전 썸네일을 표시하고 자동 재생하지 않아야 한다.
- **FR-012**: 비공개·삭제·없는 사례는 고객에게 동일한 일반 404로 처리해야 한다.
- **FR-013**: 가격, 작업 과정, 업체 소개, 개인정보처리방침 페이지를 제공해야 한다.
- **FR-014**: 고객 화면의 플로팅 견적 버튼은 시각 상태만 제공하고 폼을 열지 않아야 한다.
- **FR-015**: 관리자는 로그인 없이 관리자 대시보드와 모든 목업 관리 화면에 접근할
  수 있어야 한다.
- **FR-016**: 관리자는 새 사례를 기본 비공개 상태로 생성하고 편집할 수 있어야 한다.
- **FR-017**: 사례 텍스트·분류·정렬·공개 상태는 동일한 브라우저의 새로고침 뒤에도
  유지되어야 한다.
- **FR-018**: 사용자가 선택한 실제 이미지 내용은 현재 세션에서만 미리보기하고,
  새로고침 뒤에는 기본 목업 이미지로 복원해야 한다.
- **FR-019**: 사진 선택은 회당 최대 20장, 사례당 최대 69장을 강제해야 한다.
- **FR-020**: 사진 작업은 개별 진행·성공·실패와 실패 항목 재시도를 표현해야 한다.
- **FR-021**: 관리자는 사진 단계·순서·대표 여부·공개 여부를 변경할 수 있어야 한다.
- **FR-022**: 관리자는 사례당 최대 3개의 유효한 YouTube 링크를 관리할 수 있어야 한다.
- **FR-023**: 공개 전 제목·서비스·지역·대표 이미지·공개 사진·개인정보 확인을 모두
  검증해야 한다.
- **FR-024**: 비공개 전환은 고객 목록·상세·관련 사례에서 즉시 제거되어야 한다.
- **FR-025**: 손상되거나 구버전인 저장 데이터는 기본 목업 데이터로 복구할 수 있어야 한다.
- **FR-026**: 390px 모바일과 1440px 데스크톱에서 핵심 기능이 동일하게 제공되어야 한다.
- **FR-027**: 인터랙션은 44x44px 이상의 터치 영역, 가시적 포커스, 포커스 트랩·복귀,
  색상 외 상태 표현, 동작 감소를 지원해야 한다.
- **FR-028**: 로딩·빈 결과·미디어 실패·저장 실패·업로드 제한 상태는 원인과 다음 행동을
  한국어로 제공해야 한다.
- **FR-029**: 디자인 자료가 PRD와 충돌하면 서비스·수량·공개·화면 범위는 PRD를
  따라야 한다.
- **FR-030**: 모든 임시 대체 사항은 실제 인증·DB·스토리지·서버 검증·SEO·분석·배포로
  교체하기 위한 완료 체크리스트 문서에 기록되어야 한다.
- **FR-031**: 현재 범위에는 실제 인증, 원격 데이터 저장, 서버 API, 원격 이미지 업로드,
  결제, 예약, 견적 폼을 포함하지 않아야 한다.

### Key Entities *(include if feature involves data)*

- **Service**: 네 가지 청소 서비스의 이름, 경로, 설명, 대표 미디어와 정렬 순서.
- **Portfolio Case**: 서비스에 속한 현장 사례의 제목, 요약, 안전한 표시 지역, 공간 유형,
  작업 설명, 공개 상태, 추천 순서와 공개 시점.
- **Case Media**: 사례 사진의 단계, 순서, 대표 여부, 공개 여부, 캡션, 대체 텍스트와
  현재 세션 미리보기 상태.
- **Case Video**: 사례에 속한 YouTube 식별 정보, 제목, 설명, 순서와 공개 여부.
- **Tag**: 서비스별 공간·오염·작업 범위 필터에 사용되는 분류.
- **Publish Review**: 공개 전 개인정보 및 필수 콘텐츠 확인 상태.
- **Mock Store Snapshot**: 동일 브라우저에서 복원할 사례 데이터, 스키마 버전과 저장 시점.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 고객은 홈에서 원하는 네 서비스 중 하나를 1회 선택으로 열 수 있다.
- **SC-002**: 고객은 서비스 목록에서 사례를 열고 빠른보기로 확인한 뒤 원래 필터와
  위치로 3회 이내의 주요 행동으로 복귀할 수 있다.
- **SC-003**: 공개/비공개 혼합 데이터 검증에서 비공개 사례·사진의 고객 노출 건수는
  목록, 개수, 경로, 관련 사례 전체에서 0건이다.
- **SC-004**: 관리자는 새 비공개 사례의 최소 정보 입력과 첫 사진 선택 흐름을 모바일
  기준 3분 안에 완료할 수 있다.
- **SC-005**: 20장 초과 선택과 70번째 사진 추가는 모두 처리 시작 전에 100% 차단된다.
- **SC-006**: 저장된 목업 사례의 텍스트·분류·공개 상태는 동일 브라우저에서 새로고침
  후 100% 복원된다.
- **SC-007**: 핵심 고객 및 관리자 흐름은 390px와 1440px에서 가로 스크롤이나 잘린
  주요 행동 없이 완료된다.
- **SC-008**: 주요 메뉴·필터·모달·라이트박스·관리자 폼은 키보드만으로 완료 가능하고,
  모달을 닫은 뒤 포커스가 원래 트리거로 100% 복귀한다.
- **SC-009**: 계획된 단위·컴포넌트·사용자 흐름 검증과 정적 검사 및 프로덕션 빌드가
  모두 통과한다.
- **SC-010**: 실제 배포 전 필요한 인증·데이터·스토리지·보안·SEO·운영 교체 항목이
  단일 체크리스트에 100% 기록되고 각 항목에 완료 증거 기준이 있다.

## Assumptions

- PRD가 디자인 PDF 및 UI 프롬프트와 충돌할 때 항상 우선한다.
- 초기 콘텐츠는 개인정보가 없는 현실적인 한국어 목업 데이터로 제공한다.
- 동일 브라우저·동일 기기에서의 목업 상태 복원만 보장하며 기기 간 동기화는 하지 않는다.
- 실제 선택 파일은 브라우저 세션이 끝나면 복원할 수 없음을 관리자 UI에 명시한다.
- 가격, 로고, 연락처, 사업자 정보는 교체 가능한 샘플이며 고객 화면에 샘플 배지를
  노출하지 않고 후속 연동 문서에 기록한다.
- 실제 백엔드 단계에서 현재 저장소 계약을 유지하되 보안·검증·업로드 동작은 서버
  기준으로 다시 구현하고 재검증한다.

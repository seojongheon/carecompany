# Research: 사례 관리 중심 편집과 전후 사진 구분

## 결정 1: 생성 진입 통합

- **Decision**: 사례 관리 화면 상단에 `사례 추가`를 두고 대시보드·관리자 메뉴의 별도 생성 링크를 제거한다.
- **Rationale**: 목록 확인, 추가, 편집이 한 업무 맥락에 모여 탐색 부담이 줄어든다.
- **Alternatives**: 대시보드 빠른 생성 유지, 생성 전용 메뉴 유지.

## 결정 2: 두 개의 독립 업로드 영역

- **Decision**: 작업 전·후 패널을 동시에 노출하고 패널이 단계 값을 소유한다.
- **Rationale**: 업로드 후 일괄 분류나 탭 전환보다 단계 오지정을 줄인다.
- **Alternatives**: 단일 드롭존+단계 선택, 탭 전환형 드롭존.

## 결정 3: 라이트박스 헤더 배지

- **Decision**: 라이브러리의 `slideHeader` 확장 지점에 배지를 렌더링한다.
- **Rationale**: 이미지 위 우측 상단에 안정적으로 겹치면서 확대·캡션 기능을 유지한다.
- **Alternatives**: 이미지 자체 합성, 캡션 텍스트에 단계 추가.

# Feature Specification: 사례 관리 중심 편집과 전후 사진 구분

**Feature Branch**: `007-case-management-flow`  
**Created**: 2026-07-23  
**Status**: Approved  
**Input**: 사례 추가·편집 진입을 사례 관리로 모으고, 작업 전·후 사진을 따로 업로드하며, 확대 사진에 단계 배지를 표시한다.

## User Scenarios & Testing

### User Story 1 - 사례 관리에서 추가와 편집 (Priority: P1)

관리자는 별도의 대시보드 빠른 생성 메뉴를 찾지 않고 사례 관리 화면에서 사례를 추가하거나 기존 사례를 편집한다.

**Independent Test**: 사례 관리 화면에 `사례 추가`와 각 사례의 `편집`이 보이고, 대시보드·관리자 내비게이션에는 별도 `새 사례` 항목이 없다.

**Acceptance Scenarios**:

1. **Given** 관리자가 사례 관리에 들어왔을 때, **When** 상단을 확인하면, **Then** `사례 추가` 버튼으로 비공개 사례 생성 화면에 진입한다.
2. **Given** 사례 목록이 있을 때, **When** 행의 `편집`을 누르면, **Then** 해당 사례 편집 화면에 진입한다.
3. **Given** 관리자가 대시보드나 관리자 메뉴를 볼 때, **Then** 별도 `새 사례` 바로가기가 노출되지 않는다.

### User Story 2 - 작업 전·후 사진을 따로 업로드 (Priority: P1)

관리자는 사례 편집 화면에서 작업 전 사진과 작업 후 사진을 서로 다른 업로드 영역에 넣고, 업로드된 사진이 선택한 단계로 자동 분류되게 한다.

**Independent Test**: 각 드롭존으로 동일한 형식의 파일을 넣었을 때 생성되는 미디어 단계가 각각 `before`, `after`이다.

**Acceptance Scenarios**:

1. **Given** 사례 편집 화면에서, **When** `작업 전 사진` 드롭존에 파일을 선택하거나 끌어 놓으면, **Then** 사진은 작업 전 단계로 등록된다.
2. **Given** 사례 편집 화면에서, **When** `작업 후 사진` 드롭존에 파일을 선택하거나 끌어 놓으면, **Then** 사진은 작업 후 단계로 등록된다.
3. **Given** 파일 제한을 넘거나 지원하지 않는 파일을 넣으면, **Then** 기존 용량·수량·형식 검증과 오류 안내를 그대로 적용한다.

### User Story 3 - 확대 사진 단계 배지 (Priority: P2)

방문자는 전체 작업 사례의 빠른 상세보기와 사례 사진 라이트박스에서 사진을 크게 볼 때 우측 상단의 작은 배지로 작업 전·후를 즉시 구분한다.

**Independent Test**: 빠른 상세보기 대표 사진과 작업 전·후 라이트박스 사진에서 `작업 전`, `작업 후` 배지를 확인한다.

**Acceptance Scenarios**:

1. **Given** 작업 전 사진을 확대한 상태에서, **Then** 사진 우측 상단에 `작업 전` 배지가 표시된다.
2. **Given** 작업 후 사진을 확대한 상태에서, **Then** 사진 우측 상단에 `작업 후` 배지가 표시된다.
3. **Given** 작업 과정 또는 세부 사진을 확대한 상태에서, **Then** 잘못된 작업 전·후 배지는 표시되지 않는다.
4. **Given** 전체 작업 사례에서 빠른 상세보기를 연 상태에서, **Then** 대표 사진 단계가 작업 전·후라면 같은 단계 배지가 표시된다.

### Edge Cases

- 작업 전이나 작업 후 사진이 하나도 없어도 다른 업로드 영역과 편집 기능은 계속 동작한다.
- 한 업로드 영역의 선택 중인 파일과 기존 미디어를 합산하여 사례당 69장 제한을 적용한다.
- Storage 비활성 상태에서는 기존 정책대로 브라우저 미리보기만 제공하고 운영 저장 성공으로 오인시키지 않는다.
- 배지는 확대 이미지와 함께 이동하며 닫기·확대 버튼을 가리지 않는다.

## Requirements

### Functional Requirements

- **FR-001**: 사례 관리 화면은 `사례 추가` 진입과 각 사례의 `편집` 진입을 함께 제공해야 한다.
- **FR-002**: 대시보드, 데스크톱 사이드바, 모바일 관리자 메뉴에서 별도 `새 사례` 진입을 제거해야 한다.
- **FR-003**: 사례 편집 화면은 작업 전과 작업 후의 독립된 드래그앤드롭·로컬 파일 선택 영역을 제공해야 한다.
- **FR-004**: 업로드 영역은 생성 미디어에 해당 단계(`before` 또는 `after`)를 자동 지정해야 한다.
- **FR-005**: 두 업로드 영역 모두 기존 파일 형식, 파일당 20MB, 회당 20장, 사례당 69장 검증을 공유해야 한다.
- **FR-006**: 기존 사진 관리에서 단계 변경, 정렬, 공개·비공개, 대체 텍스트, 캡션, 삭제 기능을 유지해야 한다.
- **FR-007**: 고객 빠른 상세보기와 상세 라이트박스는 작업 전·후 사진에 각각 한국어 단계 배지를 표시해야 한다.
- **FR-008**: 배지는 사진 우측 상단에 고대비 작은 박스로 표시되고 키보드·확대·닫기 조작을 방해하지 않아야 한다.
- **FR-009**: Supabase Storage는 기존 비활성 플래그를 존중하고 활성화 전에는 원격 저장을 시도하지 않아야 한다.

### Assumptions

- `/admin/portfolio/new` 및 `/admin/portfolio/[id]/edit` 경로는 유지하고 진입 위치만 사례 관리로 통합한다.
- 작업 과정·세부 사진은 기존 사진 관리에서 계속 분류할 수 있다.
- 권장 이미지 규격은 기존 사례 사진 안내인 1600×1200px, 4:3, JPG/WebP를 사용한다.

## Success Criteria

- **SC-001**: 관리자는 사례 관리 화면에서 두 번 이하의 클릭으로 새 사례 생성 또는 기존 사례 편집을 시작할 수 있다.
- **SC-002**: 작업 전·후 드롭존으로 등록한 사진의 단계 분류가 자동화된 테스트에서 100% 일치한다.
- **SC-003**: 작업 전·후 확대 사진은 390px와 1440px 화면에서 우측 상단 배지를 식별할 수 있다.
- **SC-004**: 기존 관리자 생성·편집·공개 및 고객 사례 탐색 회귀 테스트가 모두 통과한다.

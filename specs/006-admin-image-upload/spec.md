# Feature Specification: 관리자 이미지 업로드

**Feature Branch**: `main`

**Created**: 2026-07-23

**Status**: Approved by project owner

**Input**: 사례 업로드와 랜딩페이지 대표이미지 업로드를 관리자 화면에 추가한다. 드래그앤드롭, 로컬 파일 선택, 권장 이미지 크기 안내를 지원한다. Storage 미활성 상태에서는 준비를 완료하고 활성화 후 실제 업로드로 전환한다.

## User Scenarios & Testing

### User Story 1 - 사례 사진 준비 및 관리 (Priority: P1)

관리자는 사례 편집 중 사진 파일을 드롭하거나 선택해 업로드 준비와 미리보기를 할 수 있다.

**Why this priority**: 사례 사진은 관리자 운영의 핵심 콘텐츠다.

**Independent Test**: 사례 편집 화면에 파일을 드롭하거나 선택했을 때 동일한 검증·미리보기·권장 크기 안내가 표시된다.

**Acceptance Scenarios**:

1. **Given** 사례 편집 화면, **When** 관리자가 최대 20장의 허용 이미지 파일을 드롭하면, **Then** 파일별 준비 상태와 미리보기가 표시된다.
2. **Given** 사례 사진, **When** 파일이 4:3 권장 비율과 다르면, **Then** 저장을 막지 않고 잘림 가능성 안내를 표시한다.
3. **Given** Storage가 비활성, **When** 사진을 준비하면, **Then** 실제 원격 저장 전 미리보기임이 표시된다.

### User Story 2 - 랜딩 히어로 이미지 관리 (Priority: P1)

관리자는 홈페이지 관리 화면에서 히어로 이미지를 드롭하거나 선택하고 초안 저장·게시할 수 있다.

**Why this priority**: 랜딩 대표이미지는 첫 화면의 핵심 운영 콘텐츠다.

**Independent Test**: 히어로 이미지를 선택하면 16:9 권장 크기와 대체 텍스트 입력이 보이고, URL이 준비된 콘텐츠는 공개 랜딩에 반영된다.

**Acceptance Scenarios**:

1. **Given** 홈페이지 관리 화면, **When** 관리자가 단일 이미지를 드롭하면, **Then** 미리보기와 1920 × 1080px 권장 안내가 표시된다.
2. **Given** 이미지와 대체 텍스트가 있는 초안, **When** 관리자가 게시하면, **Then** 공개 랜딩 히어로에 해당 이미지가 사용된다.
3. **Given** 히어로 이미지에 대체 텍스트가 없음, **When** 게시를 시도하면, **Then** 게시가 차단된다.

### Edge Cases

- 허용하지 않은 파일 형식 또는 20MB 초과 파일은 선택 목록에 추가하지 않는다.
- 사례당 69장 또는 회당 20장을 넘는 파일은 추가하지 않는다.
- 여러 히어로 파일이 드롭되면 첫 번째 파일만 선택하고 안내한다.
- Storage 비활성 상태에서 페이지를 새로고침하면 로컬 미리보기는 유지되지 않지만 기존 공개 목업은 유지된다.

## Requirements

- **FR-001**: 사례 및 히어로 업로드 입력은 드래그앤드롭과 로컬 파일 선택을 모두 지원해야 한다.
- **FR-002**: 사례 업로드에는 1600 × 1200px, 4:3, JPG/WebP 권장을 표시해야 한다.
- **FR-003**: 히어로 업로드에는 1920 × 1080px, 16:9, JPG/WebP 권장을 표시해야 한다.
- **FR-004**: 허용 형식·20MB 파일 한도·사례 수량 제한을 적용해야 한다.
- **FR-005**: 권장 비율 불일치는 경고만 표시하고 유효한 이미지를 차단하지 않아야 한다.
- **FR-006**: Storage 비활성 상태에서 원격 업로드 성공을 표시해서는 안 된다.
- **FR-007**: 공개 랜딩은 저장된 히어로 공개 URL이 있을 때만 해당 URL을 사용하고, 없으면 기존 목업을 유지해야 한다.
- **FR-008**: 실제 Storage 활성화 시 기존 UI를 바꾸지 않고 사례와 히어로 저장 게이트웨이를 연결할 수 있어야 한다.

## Success Criteria

- **SC-001**: 두 관리자 화면에서 클릭 선택과 드롭이 같은 파일 검증 결과를 낸다.
- **SC-002**: 사례와 히어로의 권장 이미지 정보를 관리자가 업로드 전에 볼 수 있다.
- **SC-003**: 랜딩 공개 화면은 이미지 URL 존재 여부에 따라 실제 이미지와 목업을 정확히 전환한다.
- **SC-004**: 전체 테스트, 린트, 타입 검사, Playwright 흐름, 프로덕션 빌드가 통과한다.

## Assumptions

- Supabase Storage는 별도 비용·정책 승인이 끝날 때까지 비활성이다.
- 실제 히어로 이미지 URL은 Storage 활성화 후 공개용 처리 경로에서 생성된다.
- 현재 사례 원본 파일의 최대 크기 정책은 20MB를 유지한다.
